NOTE : The widest breakpoint is always named large and the narrowest small. You can add any other breakpoint between.

Default variables

SASS variables Default Type Description
nb-of-columns 12 integer Define the number of columns
width 960px integer Define the width of the grid
gutter-width 30px integer Define the width of gutters
rowPadding 30px integer Define the padding of rows
colPadding 30px integer Define the padding of columns
rowMarginBottom 30px integer Define the margin bottom of rows
colMarginBottom 0 integer Define the margin bottom of columns
positionScheme 'inline-block' string Define a positionning scheme. Values can be 'inline-block' or 'float'
row cols string Customize class name of rows
col col string Customize class name of columns
offsetClass offset string Customize name of offset classes. Works if offset is set to true
centered false boolean Choose true to center the grid on the page
small false boolean Choose true to have a unique width of 100% on the smallest breakpoint (small)
offset false boolean Choose true if you need offset classes
visibility false boolean Choose true if you need classes that let you show or hide columns based on screen size. It creates a -hidden and -visible classes. For instance : large-hidden, medium-visible, …
dev false boolean If set to true, it adds a pink transparent background-color behind elements and blue sides border to each columns
debug false boolean If set to true, it send debugging information to the ‘Terminal’

floats or inline-block

Don’t know the difference between both ? check this page.

  • With inline-block :
    • It allows you to vertically align elements
    • Each .col goes to a new line no matter the heights of previous columns
    • Because of the whitespace, font-size is set to 0 in the parent element
